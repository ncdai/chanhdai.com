## Features

- Automatically detects mobile device support via pointer media query.
- Supports custom duration and vibration patterns on Android devices.
- Uses [Vibration API](https://developer.mozilla.org/docs/Web/API/Vibration_API) on Android and iOS checkbox trick on iOS.
- Gracefully degrades on desktop devices.

{/* ## Platform Support */}
{/* - Android/modern browsers: [Vibration API](https://developer.mozilla.org/docs/Web/API/Vibration_API) (supports custom patterns). */}
{/* - iOS: Checkbox switch element trick (fixed duration). */}

## Usage

```tsx
import { haptic } from "@/lib/haptic"
```

```tsx
<Button onClick={haptic}>Haptic</Button>
```

## Examples

### Custom Duration

```tsx
// Only work on Android devices.
<Button onClick={() => haptic(200)}>Long vibration</Button>
```

### Vibration Pattern

Accepts an array for alternating vibrate/pause patterns. See [Vibration API docs](https://developer.mozilla.org/docs/Web/API/Vibration_API) for details.

```tsx
// Only work on Android devices.
<Button onClick={() => haptic([200, 100, 200])}>Pattern</Button>
```
